## Объяснение алгоритма Кнута (Dancing Links / Algorithm X)

### Что такое задача точного покрытия (Exact Cover)?

Задача точного покрытия формулируется так: дано множество X и коллекция S подмножеств X, нужно найти подколлекцию S*, такую что каждый элемент из X содержится ровно в одном подмножестве из S* [](https://ar5iv.labs.arxiv.org/html/1010.5890).

### Как судоку превращается в точное покрытие?

Правила судоку можно представить как 4 типа ограничений [](https://github.com/zacholade/sudoku-dancing-links)[](https://github.com/pkkid/python-algorithmx/#start-of-content):

1. **Ограничение клетки** (Cell Constraint): каждая из 81 клетки должна содержать ровно одно число
2. **Ограничение строки** (Row Constraint): каждая из 9 строк должна содержать числа 1-9 ровно один раз
3. **Ограничение столбца** (Column Constraint): каждая из 9 столбцов должна содержать числа 1-9 ровно один раз
4. **Ограничение блока** (Box Constraint): каждый из 9 блоков 3×3 должен содержать числа 1-9 ровно один раз

**Итого:** 4 × 81 = 324 ограничения (колонки в матрице) [](https://github.com/zacholade/sudoku-dancing-links)

**Количество возможных вариантов:** для каждой из 81 клетки есть 9 возможных чисел, то есть 81 × 9 = 729 вариантов (строк в матрице) [](https://github.com/zacholade/sudoku-dancing-links)

### Алгоритм X

Algorithm X — это рекурсивный поиск с возвратом для решения задачи точного покрытия [](https://ar5iv.labs.arxiv.org/html/1010.5890):

1. Если матрица пуста — решение найдено
2. Выбрать колонку c с наименьшим количеством 1 (эвристика для минимизации ветвления)
3. Выбрать строку r, где A[r,c] = 1
4. Включить строку r в частичное решение
5. Удалить все колонки j, где A[r,j] = 1, и все строки, где есть 1 в этих колонках
6. Повторить рекурсивно

### Dancing Links (DLX) — оптимизация

Dancing Links — это эффективная реализация Algorithm X с использованием циклических двусвязных списков [](https://github.com/zacholade/sudoku-dancing-links):

* Каждый элемент матрицы (1) представлен узлом с ссылками влево, вправо, вверх, вниз
* Удаление узла: `node.right.left = node.left; node.left.right = node.right`
* Восстановление узла: `node.right.left = node.left.right = node` (это и есть "танец")
* Благодаря сохранению ссылок, удалённый узел может быть легко восстановлен [](https://github.com/zacholade/sudoku-dancing-links)

### Почему это эффективно?

* **O(1)** для стандартной судоку 9×9 — размер фиксирован, алгоритм работает за константное время [](https://github.com/zacholade/sudoku-dancing-links)
* Эвристика "выбор колонки с наименьшим количеством 1" (S-heuristic) минимизирует ветвление [](https://github.com/yiz-bit/dlx-exact-cover)
* Для сложных головоломок Dancing Links решает судоку за ~0.17 секунды [](https://github.com/zacholade/sudoku-dancing-links)

### Ключевая идея кода

1. **Построение матрицы точного покрытия** — каждая строка представляет вариант "в клетку (row,col) поставить число val", колонки — ограничения правил судоку
2. **Поиск с возвратом** — рекурсивно выбираем строки, покрывающие все ограничения
3. **Обратное преобразование** — из индексов строк получаем координаты заполненных клеток

Этот алгоритм считается одним из самых быстрых методов для точного решения судоку
