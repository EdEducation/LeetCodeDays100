# Идея решения задачи "Longest Substring Without Repeating Characters"

Представьте, что вы проводите "окном" по строке и ищете самый длинный участок без повторяющихся букв. Окно может расширяться и сужаться.

### Ключевая идея: Скользящее окно (Sliding Window)

Используем  **два указателя** :

* **`right`** (правый) - расширяет окно, добавляя новые символы
* **`left`** (левый) - сужает окно, когда встречаем повторение

```plaintext
Шаг 1: [a]bcabcbb     → окно "a" (длина 1)
Шаг 2: [ab]cabcbb     → окно "ab" (длина 2)  
Шаг 3: [abc]abcbb     → окно "abc" (длина 3) ← максимальная
Шаг 4: a[bca]bcbb     → встретили 'a' (уже есть), сдвинули left
Шаг 5: ab[cab]cbb     → встретили 'b', снова сдвинули
Шаг 6: abc[abc]bb     → встретили 'c', сдвинули
Шаг 7: abca[bc]b      → 'b' повторяется
Шаг 8: abcab[cb]      → 'b' повторяется
```


 **Решение через скользящее окно** :

* Каждый символ обрабатывается **максимум 2 раза** (добавление и удаление)
* Сложность: **O(n)** - линейная!


### Почему это оптимально?

1. **Жадный подход** : Мы всегда пытаемся расширить окно, пока это возможно
2. **Нет возвратов** : `right` никогда не движется назад
3. **Минимальная память** : Храним только встреченные символы

**Главная идея** : Поддерживать окно уникальных символов, сдвигая левую границу при обнаружении дубликата. Это превращает задачу с квадратичной сложностью в линейную!
